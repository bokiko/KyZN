## SECURITY

### SEC-001 — SQL injection in user search endpoint

- **Severity:** CRITICAL
- **File:** `src/routes/users.ts:47`
- **Effort:** small

The `searchUsers` function interpolates `req.query.q` directly into a SQL query string without parameterization. An attacker can inject arbitrary SQL via the search box.

**Suggested fix:** Use parameterized queries: `db.query('SELECT * FROM users WHERE name LIKE $1', ['%' + query + '%'])`

---

### SEC-002 — Missing rate limiting on auth endpoints

- **Severity:** HIGH
- **File:** `src/routes/auth.ts:12`
- **Effort:** medium

The `/login` and `/register` endpoints have no rate limiting. An attacker can brute-force credentials or create unlimited accounts.

**Suggested fix:** Add `express-rate-limit` middleware with 5 attempts per 15 minutes for login, 3 per hour for registration.

## CORRECTNESS

### BUG-001 — Off-by-one in pagination

- **Severity:** HIGH
- **File:** `src/lib/paginate.ts:23`
- **Effort:** small

`Math.ceil(total / pageSize) - 1` returns one fewer page than expected when `total` is an exact multiple of `pageSize`. Page 0 is skipped.

**Suggested fix:** Remove the `- 1`. The ceiling division already produces the correct page count.

---

### BUG-002 — Unchecked null in user profile

- **Severity:** MEDIUM
- **File:** `src/components/Profile.tsx:34`
- **Effort:** small

`user.address.city` crashes with "Cannot read property 'city' of null" when a user has no address record. The optional chain is missing.

**Suggested fix:** Change to `user.address?.city ?? 'Not set'`

## PERFORMANCE

### PERF-001 — N+1 query in dashboard

- **Severity:** MEDIUM
- **File:** `src/routes/dashboard.ts:15`
- **Effort:** medium

The dashboard loads all projects, then queries each project's task count individually in a loop. With 100 projects, this fires 101 queries.

**Suggested fix:** Use a single query with `GROUP BY project_id` and `COUNT(*)`, or use an ORM eager-loading strategy.

---

### PERF-002 — Unindexed foreign key

- **Severity:** MEDIUM
- **File:** `migrations/003_add_comments.sql:8`
- **Effort:** small

The `comments.post_id` column has no index. Every query filtering comments by post requires a full table scan.

**Suggested fix:** `CREATE INDEX idx_comments_post_id ON comments(post_id);`
